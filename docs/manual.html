畑管理アプリ アプリを開く

SPECIFICATION AND USER GUIDE

畑の記録を、毎日の判断に使える形へ。

区画ごとの栽培、収穫、費用、作業、農薬、バックアップをまとめて扱うための仕様と利用マニュアルです。スマホで日々入力し、PCでは全体を見渡せます。

目次 アプリの概要 機能一覧 日々の使い方 バックアップ 使い始める スマホに追加 PC・スマホ同期 公開・更新手順 注意点と保守
01 / OVERVIEW

アプリの概要

家庭菜園や畑の栽培情報を、苗単位で残すためのWebアプリです。左右に分かれた畑ページをマス目として管理し、収穫の数、栽培費、農薬の適用可否までを同じデータにまとめます。

STEP 1苗を配置

畑マップの区画を選び、野菜・品種・植え付け日を登録します。

STEP 2日々の記録

収穫、費用、作業、農薬散布をスマホからその場で入力します。

STEP 3全体を確認

野菜別の収穫、費用、直近の推移、農薬の安全状態を確認します。

STEP 4データを守る

バックアップ、またはSupabase同期で端末変更・故障に備えます。

通常の利用にアプリのインストールは不要です。公開URLをChromeまたはSafariで開くだけで使えます。スマホのホーム画面に追加すると、専用アプリのように起動できます。
02 / FEATURES

現在使える主な機能

記録する単位を増やしすぎず、畑仕事の流れに沿って入力できるように構成しています。

01

畑マップ・苗配置

左ページ・右ページを切り替え、行と列のマスに苗を配置。列数・段数を変更しても、既存データをできるだけ維持します。

02

野菜・品種の登録

野菜はイラスト付きの選択画面から選択。品種、植え付け日、栽培シーズン、メモを苗ごとに記録できます。

03

収穫カウント

マップ上の苗で素早く増減。編集画面では数量・単位を指定して一括加算・減算できます。

04

収穫ダッシュボード

野菜別・品種別の合計、期間別推移、収穫履歴を集計。PCでは広い表示領域を使って確認できます。

05

費用・レシート

日付、分類、金額、メモを記録。レシート画像から合計候補を読み取り、手入力で確定できます。

06

作業記録・予定

水やり、追肥、剪定などの作業履歴と、これから行う予定を野菜・苗に紐づけて管理します。

07

農薬対応表・散布記録

野菜ごとの適用情報を確認し、散布を記録。畑マップでは選択した農薬に対するOK・注意・NGを表示します。

08

栽培終了・アーカイブ

終了した苗は畑マップから外して保管。年・シーズンで絞り込み、必要なら畑へ戻せます。

09

クラウド同期

任意でSupabaseを設定すると、PCとスマホで同じデータを共有できます。設定しなければ端末内だけで使えます。

10

バックアップ・復元

JSONファイルへ全記録を保存。復元前の状態は端末内に自動退避され、すぐに取り消せます。

03 / DAILY USE

日々の使い方

最初に苗を置き、以後は収穫・作業・費用を記録するだけです。記録は自動保存されます。

最初にすること

畑マップ上の空き区画の追加操作から、野菜、品種、植え付け日を選んで登録します。苗の色とアイコンで野菜を見分けられます。

収穫を入力する

収穫モードで苗を選び、プラス・マイナスを押します。数をまとめて変える場合は苗の「編」を開き、「収穫を一括調整」を使います。

苗の情報を直す

苗カードの「編」から野菜、品種、日付、メモを変更できます。不要になった苗は栽培終了にして履歴へ移します。

費用を残す

費用タブで、日付、名前、分類、税込金額を入力します。レシートがある場合は写真を選び、読取候補を確認して登録します。

作業を残す

作業タブで、水やりや追肥などを登録します。次にすることは予定として入れると、未完了の一覧から確認できます。

農薬を確認する

農薬対応表で適用を確認し、散布記録を登録します。畑マップ上部で農薬を選ぶと、現在植えている苗の状態を色で確認できます。

農薬の扱いについて

  • 対応表は確認を補助するためのものです。使用前には必ず購入した製品ラベルの適用作物、希釈倍率、使用時期、使用回数を確認してください。
  • 散布後の収穫可否は、登録した散布日と収穫前日数をもとに表示します。表示だけで判断せず、ラベルの記載を優先してください。
  • 適用データの追加・見直しは、農薬対応表ページで行います。
04 / BACKUP

バックアップと復元

スマホの故障、ブラウザデータ削除、誤操作に備えるため、月に一度または大きな変更前にバックアップしてください。

1

保存する

アプリ右上の箱アイコンを押し、「バックアップを保存」を選択します。field-harvest-backup-日付.json が端末に保存されます。

2

復元前に確認

同じ画面で「ファイルを選択」。苗・収穫・費用などの件数が表示されるため、対象ファイルを確認します。

3

復元・取り消し

「この内容で復元」を押すと置き換わります。直前の状態は端末内へ退避されるため、その画面で取り消せます。

バックアップには畑・収穫・費用・作業・予定・農薬散布・アーカイブの記録が含まれます。SupabaseのURLやキーなど、クラウド接続設定は含まれません。
05 / GET STARTED

このアプリを使い始める手順

普段使うだけなら、インストール作業は不要です。公開URLを開く方法と、アプリ自体を管理・更新する方法は別です。

普段使う人

Google Chrome、Safari、Microsoft Edgeなどの最新ブラウザだけで利用できます。アカウント作成やソフトのインストールは不要です。

公開アプリを開く
目的必要なものすること
スマホ・PCで使うChrome / Safariなどのブラウザ公開URLを開く。記録は最初はその端末内に保存されます。
スマホへ追加スマホのブラウザ下の「スマホに追加」を実施。ホーム画面からすぐ起動できます。
PC・スマホで同じ記録を使うSupabaseアカウントクラウド同期を設定し、同じ共有データIDを両方の端末に設定します。
アプリを更新・公開するGitHubアカウント、Gitローカルの変更をGitHubへpushし、GitHub Pagesで公開します。
06 / PHONE

スマホのホーム画面に追加する

追加するとブラウザのタブを探さずに起動できます。アプリのような全画面表示で開くこともできます。

iPhone / iPad（Safari）

公開URLをSafariで開く。画面下の共有ボタンを押す。「ホーム画面に追加」を選ぶ。名前を確認し「追加」を押す。

Android（Chrome）

公開URLをChromeで開く。右上のメニューを押す。「ホーム画面に追加」または「アプリをインストール」を選び、画面の案内に従います。

更新が見えないとき

  • アプリを閉じて、ホーム画面のアイコンからもう一度開きます。
  • ブラウザで公開URLを開き直し、更新または再読み込みを行います。
  • 画面上の「最終更新」時刻が新しくなっているか確認します。公開直後はGitHub Pagesの反映に数分かかることがあります。
07 / CLOUD SYNC

PCとスマホで同じデータを使う

初期状態のデータはブラウザごとに保存されます。同じ記録を複数端末で扱う場合だけ、Supabaseによるクラウド同期を設定します。

1

Supabaseを準備

Supabaseで無料アカウントとプロジェクトを作成します。SQL Editorでdocs/supabase-schema.sqlの内容を実行します。

2

PCに設定

アプリ右上のクラウドアイコンを開き、Project URL、publishable key、共有データIDを入力します。共有IDは「生成」で作れます。

3

スマホへ設定

PC側で「設定リンクをコピー」を使い、スマホでそのリンクを開きます。最後にクラウドへ保存し、もう一方で取得します。

入力するのは browser 用の publishable key または anon public key です。secret key / service_role key は絶対にアプリやスマホへ入力しないでください。
08 / PUBLISH AND UPDATE

アプリを更新・公開する手順

機能や画面を変える管理者向けの手順です。日常利用だけなら、この作業は不要です。

最初にインストールするもの

Git for Windows、最新のブラウザ、GitHubアカウントを用意します。コードを手で編集する場合はVisual Studio Codeが便利です。Node.jsは開発用ローカルサーバーを使う場合だけ必要です。

GitHub Pagesを有効にする

GitHubでリポジトリを作成し、ファイルをmainブランチへアップロードします。Settings → Pages → Deploy from a branch → main / root を選びます。

更新を公開する

変更後、Gitでコミットしてpushします。GitHub Pagesが自動で更新されます。公開URLはGitHubの Settings → Pages に表示されます。

ローカルで確認する

ファイルを直接開くか、開発用サーバーを使って確認します。このリポジトリにはtools/start-mobile-server.cmdがあります。

公開前の確認項目

  • PCとスマホ幅の両方で、文字やボタンが重なっていない。
  • 収穫のプラス・マイナス、苗編集、費用登録、バックアップ画面が動く。
  • 更新時刻とservice workerのキャッシュ番号を更新し、古い画面が残らないようにする。
  • Supabaseを使う場合、secret keyがHTMLやJavaScriptに入っていない。
09 / CARE

注意点とおすすめの運用

記録を失わず、毎年の栽培比較に使える状態を保つための基本ルールです。

月1回はバックアップ

JSONファイルはスマホ本体だけでなく、クラウドストレージやPCにも保管します。大きな編集の前にも保存します。

同期前後は確認

複数端末で同時に編集した場合は、最後に保存した内容が優先されます。重要な編集後はクラウドへ保存してから別端末で取得します。

農薬はラベル優先

アプリの対応表は補助です。実際の使用は製品ラベル、地域の指導、作物の状態を優先して判断します。
