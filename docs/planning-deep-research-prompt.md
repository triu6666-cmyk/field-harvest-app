# 栽培計画データ調査用プロンプト

以下の `[地域]` を自分の畑の都道府県・市町村に置き換えて、ChatGPT Deep Research へ入力してください。

```text
あなたは日本の家庭菜園向け栽培暦を、一次情報中心で調査するリサーチャーです。

目的:
静的Webアプリ「畑管理アプリ」の栽培計画ページへ登録する、野菜ごとの作付時期と収穫開始目安を作る。

対象地域・条件:
- 地域: [地域]
- 栽培環境: 露地の家庭菜園
- 基準: 一般的な家庭菜園の標準的な管理。ハウス栽培・促成栽培・高冷地向けの数値は混ぜない。
- 調査基準日: 2026年

調査対象の野菜:
トマト、ミニトマト、ナス、きゅうり、ゴーヤ、ズッキーニ、ピーマン、赤パプリカ、黄パプリカ、ししとう、オクラ、いんげん、枝豆、えんどう、とうもろこし、かぼちゃ、スイカ、メロン、大根、かぶ、にんじん、じゃがいも、さつまいも、さといも、玉ねぎ、にんにく、しょうが、ねぎ、レタス、キャベツ、白菜、ブロッコリー、ほうれん草、小松菜、春菊、パセリ、シソ、アスパラ、いちご

調査ルール:
1. 都道府県の農業試験場、普及指導センター、自治体、農研機構、種苗会社の公式栽培情報を優先する。
2. 各数値は「種まきから収穫開始まで」と「苗の定植から収穫開始まで」を明確に分ける。
3. 正確な日数が示されない場合は、根拠となる月別の播種・定植・収穫期間から、日数の入力を空欄にしてよい。推測値を断定しない。
4. 同じ野菜でも品種、気温、育苗期間で大きく変わる場合は note に注意事項を書く。
5. 出典URLと、どの項目の根拠かを必ず示す。

出力形式:
最初に注意事項を短くまとめ、その後に次の JavaScript オブジェクト形式だけをコードブロックで出力する。
数値が確認できない項目は null を入れる。

window.FIELD_HARVEST_CROP_CALENDAR = {
  "トマト": {
    "seedDays": null,
    "seedlingDays": null,
    "seedSowingWindow": ["MM-DD", "MM-DD"],
    "seedlingPlantingWindow": ["MM-DD", "MM-DD"],
    "harvestWindow": ["MM-DD", "MM-DD"],
    "note": "品種・気温による注意",
    "sources": ["URL"]
  }
};

最後に、野菜ごとに「情報確認済み」「日数未確認」「地域資料なし」を一覧表にする。
```
